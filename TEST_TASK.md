# Тестовое задание для Full-Stack разработчика (Framework7 + PHP)

## Описание проекта

Необходимо разработать простое мобильное приложение **"Каталог товаров"** с минимальным функционалом.

---

## Общие требования

- **Срок выполнения**: 1-2 часа
- **Уровень сложности**: Middle
- **Формат предоставления**: Репозиторий на GitHub/GitLab с README или архив с кодом

---

## Техническое задание

### Frontend (Mobile App)

#### Технологии:
- **Framework7** (любая версия: Vue.js, React.js или Vanilla JS)
- Можно использовать Framework7 CLI для быстрого старта
- **JavaScript/TypeScript**

#### Функциональность:

1. **Список товаров**
   - Отображение каталога товаров в виде списка или карточек
   - Каждый товар показывает: название, цену и изображение (URL картинки)
   - При клике на товар - переход на страницу детального просмотра

2. **Детальная страница товара**
   - Полная информация о товаре: название, описание, цена
   - Изображение товара
   - Кнопки редактирования и удаления

3. **Создание/редактирование товара**
   - Форма с полями: название (обязательное), описание, цена (обязательное), URL изображения
   - Валидация полей
   - Кнопка "Сохранить"

4. **API интеграция**
   - Получение списка товаров с сервера
   - Получение информации о конкретном товаре
   - Создание товара через API
   - Обновление товара через API
   - Удаление товара через API
   - Обработка ошибок подключения

#### UI/UX требования:
- Использование компонентов Framework7 (ListView, Card, Button, Form, Page)
- Отображение изображений товаров (если URL указан)
- Простой и понятный интерфейс
- Индикатор загрузки при запросах к API

**Примечание**: Можно запускать в браузере для тестирования, не обязательно собирать через Capacitor/Cordova.

---

### Backend (API)

#### Технологии:
- **PHP 7/8** + **Laravel** или **Symfony** (можно использовать один из них)
- **MySQL**

#### Функциональность:

API endpoints (REST):

1. **`GET /api/products`** - получить список всех товаров
   - Ответ: JSON массив товаров
   - Формат: `[{id, name, description, price, image_url, created_at}]`

2. **`GET /api/products/{id}`** - получить информацию о товаре
   - Ответ: объект товара
   - Формат: `{id, name, description, price, image_url, created_at, updated_at}`

3. **`POST /api/products`** - создать новый товар
   - Тело запроса: `{name: "Название", description: "Описание", price: 100, image_url: "https://..."}`
   - Валидация: name и price обязательны, price должен быть числом
   - Ответ: созданный товар

4. **`PUT /api/products/{id}`** - обновить товар
   - Тело запроса: `{name: "Новое название", description: "...", price: 150, image_url: "..."}`
   - Ответ: обновленный товар

5. **`DELETE /api/products/{id}`** - удалить товар
   - Ответ: успешное удаление

#### Требования к API:
- CORS настройки (разрешить запросы с фронтенда)
- Валидация входных данных
- Обработка ошибок (404, 422, 500)
- Базовая структура ответов

#### База данных:
- Таблица `products` с полями:
  - `id` (primary key)
  - `name` (varchar) - название товара
  - `description` (text, nullable) - описание товара
  - `price` (decimal) - цена товара
  - `image_url` (varchar, nullable) - URL изображения товара
  - `created_at`, `updated_at` (timestamps)
- Можно использовать миграции или SQL файл
